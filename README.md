# 🦞 提示词盒子 (Prompt Box)

一个用来管理 AI 提示词的小工具。跨平台的提示词管理器。

## 📸 截图

| 深色模式 | 浅色模式 |
|---------|---------|
| 流畅的毛玻璃 UI，让你在写提示词的时候也觉得自己很有品味 | 阳光开朗的浅色界面，适合白天使用 |

## ✨ 功能特性

- **提示词 CRUD** — 创建、编辑、删除、复制，基础操作一个不少
- **文件夹分类** — 给提示词分门别类，支持自定义颜色
- **标签系统** — 灵活的标签管理，支持重命名、删除、合并
- **搜索** — 搜索标题、内容、标签，还支持文件夹范围切换
- **复制统计** — 记录每个提示词被复制了多少次，满足你的掌控欲
- **复制历史** — 按时间线回顾所有复制记录
- **附件支持** — 图片预览、文件存储，提示词不只是文字
- **模板系统** — 内置 5 个常用模板，也支持自定义模板
- **版本历史** — 每次保存都留下记录，误删了也不怕
- **批量操作** — 批量打标签、批量删除、批量移动
- **数据导出/导入** — JSON 备份，还能分享给别人
- **Markdown 预览** — 写的时候是编辑器，看的时候是渲染
- **{{变量}} 填充** — 支持变量占位符，使用时弹窗填入值
- **多维表格同步** — 一键同步到飞书多维表格
- **主题色** — 自由切换主题颜色，附带深色/浅色模式
- **PWA 支持** — 添加到桌面，体验接近原生 App
- **Android APK** — WebView 壳封装，状态栏/导航栏/键盘全部适配
- **更新日志** — App 内查看每个版本的改动
- **手势操作** — 左滑删除、长按菜单、多选批量

## 🏗️ 技术栈

- **前端**：纯 HTML + CSS + JavaScript（单文件，零依赖）
- **存储**：IndexedDB / localStorage
- **Android**：WebView 壳 + JavaScript Bridge
- **同步**：飞书多维表格 API

没错，没有 React，没有 Vue，没有 npm，没有 300MB 的 node_modules。就是一份 2000 行的 HTML 文件，跑起来比你的 AI 模型启动还快。

### v1.0.0 — 2026-03-20：一切的起点

故事开始于一个普通的下午，有人想要一个管理 AI 提示词的工具。不是那种臃肿的 SaaS 平台，不需要注册账号，不需要订阅，不需要同意隐私政策（虽然你肯定没读过）。

于是我们决定：写一个单文件的 HTML App。没错，一个文件，包含 CSS、JS、HTML 全部内容。

第一天就搞定了：
- 提示词的增删改查
- 文件夹分类管理
- 标签系统
- 搜索功能
- 复制历史
- 深色/浅色模式切换
- 毛玻璃 UI（毕竟好看是第一生产力）
- 模板系统
- 飞书多维表格同步

### v1.0.1 — 2026-03-21：Android 化，噩梦的开始

"能不能打包成 APK？"

这句话打开了潘多拉的盒子。

Android WebView 是一个奇妙的世界。你以为 `position: fixed` 就是 fixed？不，当键盘弹出的时候，它会飞到你意想不到的地方。你以为 `env(safe-area-inset-top)` 能解决刘海屏适配？Android WebView 说："不好意思，我不认识这个。"

踩过的坑：
- **小米 HyperOS 状态栏深色模式**：标准 API 不生效，需要小米私有 API 反射调用 `Window.setStatusBarDarkMode()`。是的，反射，Java 的反射，在 2026 年。
- **状态栏高度**：不是标准的 50px，小米 14 刘海屏是 134px。写 CSS 的时候我以为自己在给航母写样式表。
- **GitHub API 推大文件**：100KB+ 的 base64 数据要 120 秒超时，期间你只能数羊。
- **GitHub Actions 的 workflow 文件**：无法通过 API 修改，必须手动在网页操作。这一条规则浪费了我 20 分钟。

### v1.0.2 — 2026-03-22：统一模态框

在那之前，弹窗风格极其分裂——有的用 Action Sheet，有的用系统 `confirm()`，有的用 `prompt()`。用户反馈：「能不能统一一下？」

当然可以。

重命名标签、删除标签、合并标签，全部改用统一的 Action Sheet 弹窗样式。顺便把 CSS 变量 `--bg2-rgb` 补上了——之前这个变量从未定义，导致头部背景色一直没生效（你没看错，那个毛玻璃效果一直是一张空头支票）。

### v1.0.2.1 — 2026-03-22：Debug 之夜

"标签管理面板的 × 按钮显示的是 `\u00D7`，是字面意思的反斜杠 u 四个零 D 七。"

真相：`\u00D7` 是 JavaScript 的 Unicode 转义语法，只能在 JS 字符串里用。写在 HTML 正文里，浏览器只会照单全收原样显示。改成了 `&#215;` HTML 实体——就这一个字符，花了我一整个 debug 周期。

键盘自适应也在这版修好了。原理很简单：当虚拟键盘弹出时，用 `translateY(-kbHeight)` 把整个 App 容器往上推。简单粗暴，但有效。

### v1.0.2.2 — 2026-03-22：z-index 的教训

更新日志和关于面板用的是同一个 z-index（501），结果叠在一起，互相遮挡。解决方法：打开更新日志之前先把关于面板关掉。

有时候解决复杂问题不需要高深的算法，只需要加一行 `closeMopt()`。

### Android 构建

需要：
- JDK 17
- Android SDK Build Tools 34.0.0
- 一个能忍住不砸键盘的耐心

```bash
# 用 GitHub Actions 自动构建
git push origin main

# 本地签名
zipalign -f 4 app-release-unsigned.apk aligned.apk
apksigner sign --ks promptbox.keystore aligned.apk
```
